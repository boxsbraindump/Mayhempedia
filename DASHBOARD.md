# Mayhempedia 主页 / Dashboard 设计文档

> 主页专项设计。总设计见 [BRAINSTORM.md](./BRAINSTORM.md)。
> 定位：Mayhempedia = 海克斯大乱斗（ARAM: Mayhem）companion + 局内 overlay。本文只谈**主页 Dashboard**（"打开软件"阶段）。

---

## 0. 设计哲学：ARAM 助手最大的敌人是"峡谷思维"

Porofessor / Blitz 的通病 = 把召唤师峡谷排位的"数据仪表盘"思维搬到大乱斗 → 一屏 40 个数字、信息墙、赛前信息过载。

但 ARAM 玩家心智模型不同：

> **峡谷玩家是"分析师"（要赢排位）；ARAM 玩家是"半躺平但要爽"。** 不 tryhard，来放松、图乐、偶尔装 X。

ARAM 玩家要的不是"分析"，是三样：**确认**（今天打得还行吧）、**谈资**（能截图发群的身份/成就）、**省事**（别让我点太多）。

### 克制三原则（贯穿全案）
1. **一屏一主角** —— 首页第一视觉焦点只有一个（身份卡），不是六模块平铺。
2. **方向优先于数字** —— 能用箭头/颜色说清的，绝不甩两个百分比；精确值藏 hover。
3. **默认收起，渐进披露** —— 学 TFT Academy 的克制："只在需要时出现"。首页只放"确认/谈资/入口"，深度数据点进去才有。

**每个数据点自检**：它服务"确认 / 谈资 / 省事"哪一个？都不 → 砍掉，进二级页。

## 1. 三个黄金阶段的痛点，主页只管第一段

| 阶段 | ARAM 特有焦虑 | 谁解决 |
|---|---|---|
| **打开软件** | 大乱斗**无排位、无段位** → 没有"我算不算强/今天状态"的锚点，**身份认同真空** | **← 主页主战场** |
| **选人 12 秒** | 随机英雄 + 板凳 + 平衡数值不熟 → 这英雄强不强/怎么玩 | overlay |
| **进游戏** | 海克斯三选一、出装、poke | overlay + CV |

**结论：主页不抢 overlay 的活。** 选人/局内焦虑交给浮层。主页专治"打开软件"独有的**身份真空**。

### 主页信息层级（F 型视线）
1. **第一眼（左上黄金区）**：**乱斗段位/身份卡** —— 3 秒回答"我强不强"。主角。
2. **第二眼**：**版本变动速览** —— "谁被 buff 了"，回答"该期待抽到谁"。是**精选变动，不是全 tier list**。
3. **第三眼**：近期对局 + 新解锁趣味标签（谈资/分享钩子）。

## 2. 核心功能的"细节魔鬼"

### 2A · 大乱斗隐藏分/段位（官方不给 MMR，我们估）

**关键事实：**
- Riot 公开 API **不暴露 ARAM 隐藏 MMR**，别假装有。
- **你自己的对局数据 LCU 本地可拿**（`/lol-match-history/v1/...`，queueId=**450**）——**免 API key、免后端、不碰云。**
- **对手/队友峡谷段位**（做 lobby 强度代理）才需 Riot 云端 API（key + 后端 + 限流缓存）。

> **落地策略：核心分用 LCU 本地算（开箱即用、零成本），lobby 加权做成"可选增强"。** 这一刀让隐藏分从"要搭后端"变"开箱即用"。

**多维公式（示例，权重可调）：**
```
乱斗战力 ARP =
   40% × 胜率分   (近 30 场 ARAM 胜率 → 0–100)
 + 25% × 个人影响 (KDA×0.5 + 伤害占比×0.3 + 承伤/治疗占比×0.2，队内百分位)
 + 20% × 状态分   (近 10 场趋势斜率 + 连胜奖励，抓"手感")
 + 15% × 对局质量 (本局 9 人峡谷平均段位；拿不到就均值填充，可选)
```

- **段位命名走"装 X"路线**（别用青铜白银，太峡谷）：
  `深渊路人 → 冰封新秀 → 桥头霸主 → 无双乱斗手 → 冰海领主 → 大乱斗之神`（顶端 1% 动态特效边框）。
- **合规 & 诚实**：徽章标「非官方估算 · 基于近期对局」。既避免误导，"这怎么算的"本身是传播点。
- **装 X = 增长引擎**：段位一键生成**分享卡**（段位 + 招牌英雄 + 最猛一场 + Mayhempedia 水印）。

### 2B · 平衡参数 Tier List：不挡视线地展示 +X%/−Y%

**批判**：把「造成+12% / 承受−10% / 治疗−5%」并排塞进每一行 = 信息过载，扫榜根本不读数字。

**克制方案（三层渐进）：**
1. **榜单默认态：只给"方向 + 程度"，零数字。** 头像右下角**楔形角标**：颜色=方向（青蓝 buff / 暗红 nerf / 无标 原始），深浅大小=程度。一眼扫完全榜、不挡脸不堆字。
2. **hover / 点开**：才出精确值，用**双向条（diverging bar）**——中线基准，右青=增益、左红=减益，条长=幅度。比三个裸数字直觉十倍。
3. **tier 分档（S/A/B）与平衡修正解耦**：tier 是结论，平衡修正是"为什么在这个 tier"的解释，藏展开层。别让玩家自己算"+12%=几个 tier"。

> **扫榜只需"方向"，决策才需"数字"。默认给方向，按需给数字。**

### 2C · 5 个趣味硬核标签（含后台判定）

全部基于**你自己/本局公开数据**（Live Client Data + LCU match history），不碰对手隐藏信息，合规。

| 标签 | 触发判定（数据逻辑） | 分享钩子 |
|---|---|---|
| 🎯 **开局对狙王** | 前 3 分钟对英雄 poke 伤害=全队第一 且 期间死亡=0 | "还没交锋我已经赢了" |
| 🌨️ **雪球脱手** | 团队经济领先 ≥3000 后仍输（时间轴 gold diff 由正转负） | 自嘲，最易被队友 @ |
| ☄️ **一发入魂** | 单个技能/大招命中 ≥4 名敌方英雄 | 硬核操作截图 |
| 🕺 **冰河世纪の舞** | 单局泉水/等待重生累计跳舞/emote ≥60s（彩蛋） | 摸鱼梗、社群文化 |
| 💎 **满配美学家** | 最快 6 神装 或 集齐一套"棱彩三连"海克斯 | "有钱人的浪漫" |

设计要点：**解锁一个才弹**（别一次砸 20 个 = 又过载）；每个自带分享卡模板。

## 3. 自动化设置：能帮玩家做什么、不踩"外挂"红线

**判定原则（Riot 对 LCU 的容忍边界）：**
> 只读优先；写操作只限"省点击、不影响他人、不超人类速度、不给竞技优势"的便利。一旦开始**和真人抢资源**或**非人速度操作**，就从"便利"滑向"bot 行为"。

| 功能 | 风险 | 判断 & 逻辑 |
|---|---|---|
| 自动接受对局 (auto-accept) | 🟢 安全 | 纯省点击、不影响他人，广泛使用。默认关可开。 |
| 锁定后**按所选流派自动导入符文/出装** | 🟢 安全 | 只写自己符文页，零竞技优势。我们流派数据的天然落地。 |
| 记住上次设置 / 客户端省资源 | 🟢 安全 | 本地行为。 |
| 召唤师技能键位预设 | 🟢 安全 | 只设自己的，低风险。 |
| 自动宣告(hover)英雄 | 🟡 灰区 | 轻微"代操作"，做成可关。 |
| 替补席**抢**英雄 | 🔴 不做全自动 | 和真人抢板凳 = 超人类 race + 不公平优势，最像 bot。**只做"一键抢"（按键触发一次执行）**，不做"检测到自动秒抢"。 |
| 任何游戏内自动操作（走位/放技能/自动 poke） | 🔴 绝对红线 | 明确外挂，直接封。 |

**呼应既定结论**：早已判定"选人副官（按胜率帮选/reroll）是伪需求——大家玩想玩的"。自动抢板凳同理：**价值低 + 风险高**。
→ 自动化板块哲学 = **"省点击，不代打"**：默认全关、每项标风险、一键胜过全自动。

## 5. 落地前置 & 数据依赖

| 模块 | 数据源 | 是否需后端 | 前置 |
|---|---|---|---|
| 乱斗身份卡（核心分） | **LCU 本地** match history (queueId 450) | ❌ 免 key | **需先把 LCU 接进渲染层（装 Electron + 通 LCU）** |
| lobby 质量加权（可选） | Riot 云端 API（对手段位） | ✅ 要 key + 缓存 | 后置增强 |
| 版本变动速览 | Community Dragon 平衡参数（已可抓） | ❌ | 需接 ARAM 平衡数值数据 |
| 趣味标签 | Live Client Data + LCU match history | ❌ | 局内/赛后判定逻辑 |
| 分享卡 | 本地渲染 (html2canvas 等) | ❌ | —— |

> **关键前置**：身份卡是主页主角，但它依赖 **LCU 本地对局数据** → 必须先完成"装 Electron + 通 LCU"那步，主页才有真数据。在那之前可用假数据占位做 UI。

## 6. 克制承诺（验收清单）
- [ ] 首页只有一个主角（身份卡）；tier/海克斯/对局详情都在二级页。
- [ ] "变动"代替"全量"（版本区只给变了的，不给 173 个）。
- [ ] 方向默认、数字按需（角标 + hover）。
- [ ] 自动化省点击不代打（默认全关、标风险）。
- [ ] 每个数据点服务"确认/谈资/省事"之一，否则砍。
